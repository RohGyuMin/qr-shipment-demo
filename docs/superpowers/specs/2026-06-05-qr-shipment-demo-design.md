# QR 출고관리 데모 — 설계 문서

작성일: 2026-06-05
목적: 연포장 제조업체 출고관리 시스템 **1단계 데모**. 클라이언트 신뢰 확보용 동작 데모를 만들어 공유 링크 전달.

## 1. 배경

- 클라이언트: 대기업 계열사 연포장 제조업체 (필름 포장재 롤 납품).
- AS-IS: 화물차 기사 수기 작성 → 단톡방 사진 → 거래명세서/시험성적서 수동 작성, 재고 DB 없음.
- TO-BE 1단계: 팔레트 QR 스캔 → 모바일 입력폼 자동 pre-fill → 추가 항목 수동 입력 → 관리부서 4곳 실시간 대시보드 조회 (필터 + 엑셀), 앱 설치 없이 QR/링크로 접속.
- 본 문서는 **데모 범위(풀 플로우 동작)** 만 다룬다. 2단계(재고 QR 연동, 시험성적서 자동화, SAP 연동, 내부 서버 이전, 통계 고도화)는 범위 외.

## 2. 결정 사항 (확정)

| 항목 | 결정 |
|------|------|
| 작업 위치 | 새 프로젝트 디렉토리 `qr-shipment-demo`, GitHub `RohGyuMin/qr-shipment-demo`로 푸시 |
| 데모 범위 | 풀 플로우 동작 데모 (스캔 → 입력 → 대시보드) |
| QR 처리 | 알려진 8개 항목을 자체 인코딩한 샘플 QR을 직접 생성. 실제 파싱 규칙 확정 시 `decodeQr` 함수만 교체 |
| 스택/배포 | Next.js (App Router) + Vercel (HTTPS 자동) |
| 데이터 | 가벼운 서버 DB (실시간 공유 필수) |
| 인증 | 없음 (QR/링크만으로 접속 — "앱 설치 없이 바로 접속" 핵심 메시지) |
| 대시보드 | 부서 구분 없는 공용 대시보드 1개 |

## 3. 데이터 항목

### QR 자동 입력 (8개, 읽기전용 pre-fill)

관리번호, 품목, 업체명, 재질, 수량(ROL), 원단번호, 제조일자, 발주일자.

라벨 실측 예시: 관리번호 `26507KR101`, 품목 `연포장_내포장지`, 업체명 `율림P&P`, 재질 `PP`, 수량 `2.9 ROL`, 원단번호 `12`, 제조일자 `2026.04.30`, 발주일자 `2026.05.07`.

### 수동 입력 (12개)

| # | 항목 | 형태 | 비고 |
|---|------|------|------|
| 1 | 출고일시 | 날짜+시간 | 기본값=현재 시각 (수정 가능) |
| 2 | 출고량 (ROL) | 숫자 | QR 수량과 비교 |
| 3 | SAP 코드 | 텍스트 | SAP 연동 대비 |
| 4 | 출고구분 | 선택 | 정상 / 반품 / 샘플 / 추가 |
| 5 | 납품처(거래처) | 텍스트/선택 | 식품회사명 |
| 6 | 차량번호 | 텍스트 | |
| 7 | 기사명 | 텍스트 | |
| 8 | 출고 담당자 | 텍스트/선택 | 현장 작성자 |
| 9 | 팔레트 수 | 숫자 | |
| 10 | 클레임 여부 | 토글 | 예/아니오 |
| 11 | 클레임 내용 | 텍스트(긴) | 클레임=예일 때만 노출 |
| 12 | 비고 | 텍스트(긴) | 자유 메모 |

## 4. 전체 구조

Next.js 단일 코드베이스 → Vercel 배포.

화면/엔드포인트:
- `/scan` — 모바일 QR 스캐너 (카메라, 전체화면). 카메라 미지원 대비 수동 진입 링크 포함.
- `/entry` — 출고 입력폼 (QR 8항목 읽기전용 pre-fill + 수동 12항목).
- `/dashboard` — 관리부서 공용 조회 (테이블 + 날짜/품목 필터 + 합계 + 엑셀 다운로드 + 5초 폴링).
- `/api/shipments` — 출고 데이터 CRUD (GET 목록 / POST 등록).
- `/api/qr-samples` 또는 `/qr-samples` — 데모용 샘플 QR 3종 PNG 표시/생성.

데이터: Vercel Postgres 우선, 데모 단순화가 필요하면 SQLite/KV 대체 가능. 한 화면에서 입력 시 다른 화면에서 폴링으로 즉시 반영.

## 5. 데이터 흐름

```
[QR 라벨] → /scan (카메라 QR 디코드)
   → decodeQr(raw): 8개 항목 추출
   → /entry 로 이동, 폼 pre-fill
   → 현장 담당자 12개 수동 입력 → 저장
   → POST /api/shipments → DB insert
   → /dashboard: GET /api/shipments (5초 폴링) → 실시간 목록
```

- `decodeQr(raw)`: 데모 포맷(JSON 또는 `|` 구분자) 파싱. **격리 설계** — 실제 규칙 확정 시 이 함수만 교체, 나머지 코드 무영향.
- `encodeQr(fields)` + QR 생성: 샘플 QR 3종(서로 다른 품목/업체) 생성해 실물 폰 스캔 시연.

## 6. 데이터 모델

`shipments` 테이블 1개: QR 8필드 + 수동 12필드 + `id`(PK) + `created_at`. 정규화 없이 단일 테이블 — 데모 단순성 우선.

## 7. 에러 처리 / 엣지 케이스

- QR 디코드 실패 → "인식 실패, 다시 스캔" 안내.
- 카메라 권한 거부 → 안내 + 수동 진입 링크.
- 필수값 누락 → 폼 검증 (저장 차단 + 필드 표시).
- 클레임=예인데 내용 비어있음 → 검증.
- DB 오류 → 토스트 알림 (데모 수준).

## 8. 테스트 전략

- `decodeQr` 파싱 단위 테스트 — 포맷 교체 안전성 확보.
- 엑셀 export 데이터 정합성 단위 테스트.
- 핵심 플로우 수동 검증 — 실제 폰으로 샘플 QR 스캔 1회 + 대시보드 반영 확인.

## 9. 견적 / 일정 (참고)

- 1단계 90만원, 계약금 30% / 잔금 70%, 작업기간 영업일 3~4일.
- 데모는 계약 전 신뢰 확보용 → 빠른 완성 + 공유 링크 우선.

## 10. 범위 외 (2단계)

재고 생성 QR 연동, 시험성적서 자동 작성(엑셀 양식 연동), SAP 연동, 내부 서버 이전, 통계 고도화.
